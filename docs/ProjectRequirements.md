# Campus Bulletin Board System 项目要求


## 一、项目概述

**项目名称**：Campus Bulletin Board System (校园论坛)

**项目描述**：一个面向大学校园的在线论坛系统，提供用户注册、发帖、评论、点赞等功能

## 二、大程要求

组得分 = 教师打分 * 50% +（各组长打分和）/ 组数 * 50%

各组长打分在Review时进行

教师打分包括两部分：汇报分（主要依据汇报效果以及提交物质量）+ 过程分（主要依据Git评估开发过程的合理性，如开发流程是否规范、AI使用是否规范） 

组员得分 = 组员贡献率 * 该项组得分 * 组人数

组员总得分 = 各项组员得分 * 各项占比

需求分析30%、项目计划10%、系统设计20%、原型系统20%、系统测试20%


## 三、子系统拆分

### 3.1 用户与认证子系统

**职责**：完成用户身份管理与访问控制。  
**核心功能**：
- 用户注册、登录、登出
- 密码加密存储与重置
- JWT 鉴权与会话管理（可结合 Redis）
- 用户角色与权限控制（普通用户/管理员）

### 3.2 帖子与分区子系统

**职责**：管理论坛分区与主题帖全生命周期。  
**核心功能**：
- 分区（板块）创建、编辑、删除、展示
- 发帖、编辑、删除、置顶、加精
- 帖子详情与列表查询（分页、排序）

### 3.3 评论与互动子系统

**职责**：承载帖子下的互动行为。  
**核心功能**：
- 评论发布、回复、删除
- 点赞/取消点赞（帖子、评论）
- 互动状态聚合（点赞数、评论数）
  
### 3.4 通知与消息子系统

**职责**：向用户传递关键事件。  
**核心功能**：
- 评论通知、回复通知、点赞通知
- 系统公告与站内信
- 未读计数与已读状态管理

### 3.5 搜索与推荐子系统

**职责**：提升内容发现效率。  
**核心功能**：
- 帖子标题/正文关键词搜索
- 热门帖子排序（按热度、时间）
- 个性化推荐（可作为迭代目标）


### 3.6 管理后台子系统

**职责**：提供平台运营与维护入口。  
**核心功能**：
- 用户管理（封禁、解封、权限调整）
- 板块管理、公告管理
- 举报处理面板与审核统计
- 操作审计与日志查看


## 四、组内分工

### 4.1 组长

- 搭建开发环境，初始并管理 Git 仓库
- 负责定义数据库表结构
- 负责分支合并时的代码审核
- 负责项目文档的撰写与维护，包括 API 文档

### 4.2 前端（两人）

**前端A**面向用户端，负责实现用户注册、登录、发帖、评论等核心功能的界面与交互。

**前端B**面向管理端，负责实现用户管理、板块管理、公告管理等后台功能的界面与交互。

### 4.3 后端（两人）

**后端A**负责基础信息的管理。涉及子系统 3.1、3.2、3.6

**后端B**负责互动与搜索。涉及子系统 3.3、3.4、3.5

### 4.4 测试兼文档（一人）

负责系统的测试工作，包括编写单元测试以及CI/CD配置文件。
同时负责项目文档的撰写与维护，包括需求文档、设计文档、开发规范等.
